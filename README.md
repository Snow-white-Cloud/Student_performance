# Использованные решения (особенности):

## Хранение данных.

Вместо одной таблицы предоставлено разделение на `Students` и `Grades`. Данное решение обусловлено:
1) нормализацией до усиленной 3НФ или же "Нормальной формой Бойса-Кода", которая является эталоном проектирования БД;
2) стремлением уменьшить объём расходуемой БД памяти.

Было решено добавить автоинкремент `id` в качестве первичного ключа для студентов вместо уникального сочетания ФИО+группа, чтобы ускорить поиск. В противном случае для `Grades` пришлось бы хранить составной внешний ключ, что привело бы к усложнению таблицы, увеличению её объёма, замедлению поиска и дублированию данных. Однако, исходя из предоставленных данных, сочетание ФИО+группа действительно является уникальным, иначе они не имели бы смысла, поэтому на соответствующие поля наложено дополнительное ограничение на уникальность. Такой подход вызвал у меня недоумение, так как в жизни вполне возможны случаи полных совпадений ФИО и групп у двух разных людей.

Тип ФИО - `VARCHAR(100)`: 100 взято приблизительно, хотя при разработке настоящей системы, а не тестового задания, стоило бы изучить статистику по среднему количеству букв в полных инициалах. ФИО хранится цельно, так как по заданию может восприниматься неделимым элементом, тем самым не нарушая 1НФ. К тому же цельное хранение облегчает логику поиска.

Группа задана типом `CHAR(4)`, так как в исходных данных все номера заданы строго тремя цифрами и одной буквой.

Было решено добавить автоинкремент `id` в качестве первичного ключа для отметок, так как ни один иной атрибут или их сочетание не определяли запись однозначно.

## Использование SQL

В данной работе логика получения аналитики вынесена в отдельные SQL функции для демонстрации знаний. `grades_more_than` считается мною "эталонной", т.е. наиболее оптимизированной, когда как `grades_less_than` служит пособием, демонстрирующим "как можно, но не нужно". Аналитика представлена функциями для возможности в кастомизировать запросы: узнавать не только об отметках "2", но и любых других. Требуемые заданием значения передаются по умолчанию. Однако ж достаточным вариантом являлось бы написание несложных запросов, скрытых в теле функций, без возможности пользователем ввести свои предпочтения.
